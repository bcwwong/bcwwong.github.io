    Deploying mkdocs pages to GitHub Pages

    https://githowto.com/more_setup

    1. Set up account in github.com
      • username: bcwwong
    2. Create repository with the same name as username: bcwwong.github.io on GitHub. See https://pages.github.com/.
    3. Install Git on local computer.
    4. Create folder <username>.github.io under mkdocs project_folder (e.g. blog/bcwwong.github.io)
    5. cd to project_foler/<username>.github.io e.g blog/bcwwong.github.io.
    6. Initialize git: 
      git init 
git config user.email "busywong@gmail.com"
git config user.name "Brian Wong"
git remote add origin https://github.com/bcwwong/bcwwong.github.io.git
      From https://githowto.com/more_setup For Windows: 
      git config core.autocrlf true
git config core.safecrlf warn
      For Linux/Mac: 
      git config core.autocrlf input
git config core.safecrlf warn

    7. Build and deploy mkdocs pages to GitHub pages: 

      mkdocs gh-deploy --config-file ../mkdocs.yml --remote-branch main

      Warning

      May need to run git pull command first if deploying to the same GitHub page from different machines. 

      git pull
mkdocs gh-deploy --config-file ../mkdocs.yml --remote-branch main

    8. Pages are deployed to https://bcwwong.github.io/

      Install gh cli for Linux

      Need to login GitHub first. Follow instructions from https://docs.rockylinux.org/gemstones/git/00-gh_cli_installation/ to install gh cli.

      curl -fsSL https://cli.github.com/packages/rpm/gh-cli.repo | sudo tee /etc/yum.repos.d/github-cli.repo
sudo dnf -y install gh
      Then run gh auth login and follow on-screen instructions.
